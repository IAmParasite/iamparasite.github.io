B. Two Cakes

It’s New Year’s Eve soon, so Ivan decided it’s high time he started setting the table. Ivan has bought two cakes and cut them into pieces: the first cake has been cut into a pieces, and the second one — into b pieces.

Ivan knows that there will be n people at the celebration (including himself), so Ivan has set n plates for the cakes. Now he is thinking about how to distribute the cakes between the plates. Ivan wants to do it in such a way that all following conditions are met:

1.Each piece of each cake is put on some plate;
2.Each plate contains at least one piece of cake;
3.No plate contains pieces of both cakes.
To make his guests happy, Ivan wants to distribute the cakes in such a way that the minimum number of pieces on the plate is maximized. Formally, Ivan wants to know the maximum possible number x such that he can distribute the cakes according to the aforementioned conditions, and each plate will contain at least x pieces of cake.

Help Ivan to calculate this number x!

Input

The first line contains three integers n, a and b (1 ≤ a, b ≤ 100, 2 ≤ n ≤ a + b) — the number of plates, the number of pieces of the first cake, and the number of pieces of the second cake, respectively.

Ouput

Print the maximum possible number x such that Ivan can distribute the cake in such a way that each plate will contain at least x pieces of cake.

分析题目

我们最朴素的算法就是列举出所有长度为k的数列，再乘以他的妙妙趣和值，这种复杂度我们显然接受不了。

简单的思考以后，可以将数列从小到大排序，得到一个有序的序列。

我们尝试是否能算出对于一个固定的妙妙趣和值x的所有序列的个数呢？经过尝试我们很难统计这样的个数，在思考的过程中可以很容易确定妙妙趣和值大于等于x的序列个数，记为dp(x),我们可以一次自己计算出 ans += x * (dp(x) - dp(x-1))，容易想到这样是对的。这时x要从大往小枚举

我们再进行一些优化：

例如计算 1 * a1 + 2 * a2 + 3 * a3 + 4 * a4 的值，等同于 a1 + (a1 + a2) + (a1 + a2 + a3) + (a1 + a2 + a3 + a4)

这样我们可以从小往大遍历，dp(x) 从大往小一次减小，可以想到 x = 1 的情况为 a1 + a2 + a3 + a4，x = 2 的情况为 a1 + a2 + a3 以此类推

用dp[i][j]表示前i个数字中长度为k的序列，这个序列要满足的一定条件，即x+a[li]<=a[i]，li为满足不等式的最大值。递推方程式为dp[i][j] = dp[i-1][j] + dp[li][j]

递推方程式由两个部分由来：采取第k位和不采取第k位。不采取第k位显然满足条件，采取第i位要确定满足条件的下标。这样我们取得的dp[n][k]必然是满足妙妙趣和值大于x的数列个数

我们有 li 满足 a[li] + x <= a[i]，li是满足这个不等式的最大值

注意到： 当a[i] - x 的差值不变的时候， 我们a[li]的值是否发生变化呢？

答案是否定的！！！对于不同的a[i] 、 x 他们之间有着关系： 即a[i] - x为确定值时，我们有相同的li，根据这个思路我们可以用O(n) 来代替上述近似于O(n^2) ~ O(n^3) 的递归算法，极大地降低我们的运算时间。

分析题目

这个题很明显树形dp+dfs做

我没学过树形dp怎么办…其实树形dp就是和普通的dp一样，只不过采用了递归解法来求解问题

首先可以想到的是，这是一颗二叉树。当遍历到一个节点只有一个孩子的时候，这个孩子可能是左子树，也有可能是右子树。可以想到的是，调换左右子树的位置对结果没有影响。那么我们将这种情况归结为右子树为空，只遍历他的左子树即可

首先将1作为根节点，遍历dfs他的左右孩子。当dfs到一个节点没有左右孩子的时候，即叶子节点，我们可以返回一定的值（肯定要获取一定的信息啦）。
当dfs到一个结点的度为1时，遍历他的左子树即可。

我们用cor[num][0] cor[num][1] 分别表示正确输出0和正确输出1（理解为输出0或1并且正确）

我们用cor[num][0] cor[num][1] 分别表示错误输出0和错误输出1（理解为输出0或1并且错误）

left 表示左孩子的编号， right 表示右孩子的编号。

可以首先想到的是答案为 err[num][0] + err[num][1]， 再对其取模即可

当一个节点左右子树都存在的时候，我们分析一个比较简单的情况:

输出0但是错误，即理论上生成1，实际生成0

题面

在一场竞赛中有n道题目，每道题都会对应一个分值，你可以选择任意一道题作答，比如你选择了x分的题目，那么在作答完毕（假设一定可以得分）后，你将获得x分，但是这场比赛中分值等于x−1和x+1的其他题目就会消失，那么这场比赛中你最多可以得到多少分？

输入

第一个数为题目总数n（0<n<1e5)
接下来为n个整数a1，a2，a3…..(0<ai<1e5)

输出

输出你可以得到的最高分

输入样例

输出样例

分析题目

一个拐了弯的递推题目吧，可以理解为背包

首先如果你选了一个分数为x的题目，显然你需要将分数为x的所有题目全部做完，才能获得最高的分数

然后思考dp[n]表示什么：

1.n表示做完前n个题目能得到的最大值：显然不行，无法判断哪些是做过的，哪些是没有做过的

2.n表示做完所有n个不重复的物品（贪心优化），可以发现已经很接近答案了，可以再途中顺便排序，用一个结构体储存他的题目分数和所有分数加起来的分数（w[i]）。这么做似乎可行，列出递推方程式：（用before记录dp[n-1]采取的题目分数） 然而这样又出现了新的问题，尽管有before来实现所谓的筛除，但是无法清除所有记录，因为我们无法知道dp[n-1]是否采取了第n-1个题目
3.n表示做完最大值为n的获得的分数的最大值，递推方程式：
dp[n] = dp[i-1], dp[i-2]+num[i]*i.

具体怎么想呢，实际上是找变量之间的关系。题目中对分数做了限制，那么我们就在分数上下手，直接用f[n]表示最大分数为n是所获得的最大分数
