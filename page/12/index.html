C1-Zexal的过河

Posted on 2019-10-12 | In 第一次上机题解 |

题面

Zexal打算借助河中间的石砖过到河对岸去。Zexal从第一块石砖出发，接下来他可以走到第二块石砖或第三块石砖，有时候走的很不爽，甚至可以直接跨过两个石砖，到达第四块石砖，但是不能连续两次这种操作，因为这样消耗体能比较大。现在假设河中含n块石砖，且这些石砖呈直线分布，请你计算出Zexal从第一块石砖出发有多少种安全的过河方法。

输入

输入将由多组测试数据组成，以EOF结尾。

每组数据只有一行，为河中的总石砖数n（0＜n≤50）。

输出

对于每组数据，输出一行，为过河的方法数。

输入样例

输出样例

样例解释

1：一步走完；
2：先走到2再走完，或者直接走完；
3：111或12或21或3。

分析题目

我们想要走到第n步，有3中方法，从f[n-1], f[n-2], f[n-3] 走过来。

但是我们如果只用一个数组无法判断是否连续走了两步， 所以我们用两个数组来保存f1[n] 表示走到第n步且最后一步不是走3的方法数，f[2] 表示走到第n不且最后一次是走3步的方法数。我们可以得到递推公式：

f[n] = f1[n] + f2[n];

f1[n] = f1[n-1] + f1[n-2] + f2[n-1] + f2[n-1];

f2[n] = f1[n-3];

C1-点灯

Posted on 2019-10-12 | In 第一次上机题解 |

题面

有n个灯，编号0∼n−1，一开始都是关闭状态。

每次操作会拨动一个区间[L,R]灯的开关，也就是说，对于灯i，L≤i≤R，如果i是关闭状态，则操作会使灯亮，反之会使灯灭。

请问k次操作后有多少灯亮着。

输入

多组输入数据

每组数据第一行两个数n,k（1≤n≤1e9,1≤k≤1e5）

接下来k行，每行两个数l,r（0≤l≤r≤n−1）

输出

每组数据一行一个数，表示最后灯亮的个数

输入样例

输出样例

分析题目

首先分析题目。

这个题实质上是在每个给定区间加一再分析是否为奇数即可。

广大观众老爷肯定理解过差分这么一个东西，大家可以回忆一下差分的具体实现。用一个空间为n的数组储存每一位的值，计算位于位之间的差值。读取k个区间加一，左端差分+1，右端坐标加+1 同时差分减一。从1遍历到n，计算出每一位的值，这就是普通的差分了。但是在这个题里面，我们且不说我们无法存储1e9的数组，但是这1e9次计算就能让你爆的体无完肤。我们只好在比较小的k上“动脑筋”。

分析一下整个题目，发现在浩瀚无垠的1e9里，只有一小部分，即2* k个点需要改变差分的值，其他的点都会重复。那么这就是我们“离散化区间”的精髓了：我们保存下来需要改变的值，将他们存在数组[num]里面，从小到大排序，去掉重复的点（一个点处理一次就可以啦）减小k的值。从0（1）开始遍历，需要改变的点就改变他的差分的值，再加上两个num数组之间的差值，就可以计算出这一段区间的数值了。具体看代码哦
