分析题目

用dp[i][j][0]和dp[i][j][1]分别表示排列i个女生j个男生时末尾为男生的情况。

多维动归显然不行，我们注意到，连续不能超过x个女生的情况，只需要在男生后面依次“填充”min(i,x)个女生即可，根本不需要数组记录末尾连续的人数，同理男生也是一样，在女生后面“填充”min(j,y)个男生即可。

想想怎么样可以再优化一下呢？我们不妨用笔计算一下：

假设min(i,x) = 3， min(j,y) = 4

计算i = 5, j = 6 情况的时候你可以注意到
dp[5][6][0] = dp[4][6][1] + dp[3][6][1] + dp[2][6][1]

dp[5][6][1] = dp[5][5][0] + dp[5][4][0] + dp[5][3][0] + dp[5][2][0]

同样的进行下一次遍历(j++)的时候，计算i = 5, i = 7 的时候

dp[5][7][0] = dp[4][7][1] + dp[3][7][1] + dp[2][7][1]

dp[5][7][1] = dp[5][6][0] + dp[5][5][0] + dp[5][4][0] + dp[5][3][0]

我们可以发现规律：当i 相同的时候，以男生结尾的情况有重叠的部分，且相加的个数好像是确定的，个数就是min(j,y)个。同样当j相同的时候，以女生结尾的部分也是重复的，只要维护两个数组sum_1[i]，sum_2[j]来表示需要相加的“前缀和”，得出答案再更新，就可以再O(n^2)完成。TLE的点肯定是，x，y比较大的时候，这时的算法效率为O(n^3)，很容易被卡超时。

C3-Zexal的浩瀚星辰

Posted on 2019-11-01 | In 第三次上机题解 |

题面

Zexal 想要发射火箭，但是由于能源供应不足了，所以一些火箭需要延迟发射。 每个火箭每延迟一小时发射都会相应的损失。Zexal了解到，一共有n个火箭，其中第i个火箭原计划在第i小时发射，即1n时间段发射，现预计k小时后电力可以恢复正常，即所有火箭将在k+1k+n时间段内发射，

新的火箭发射计划不要求按照最初的发射计划顺序，唯一的要求是每个火箭都不能早于原定时间发射。请你帮忙计算一下最小的损失吧。

注意：时间均以小时为最小单位。由于条件有限，一次只能发射一枚火箭。

输入

输入包含多组数据。

每组数据第一行为正整数n和k（1≤n≤500000,1≤k≤500000），为火箭总数和延迟时间。

接下来是n个正整数 pi，代表第i个火箭每延迟一小时的损失费（1≤pi≤104）。

输出

对于每组数据，输出一行，为最小的损失费用。

输入样例

输出样例

分析题目

我们可以将这个题分成两种情况： n >= k 和 n < k

当 n >= k 时，即所有火箭必定延误，我们的直觉告诉我们，将延误代价最高的火箭先发射，是不是能让损失最小呢？我们又发现，我们需要计算他们发射时间之间的差值，也就是说，延误的代价还与发射时间有关，那么我们这样做是否是正确的呢？如果不正确，我们应该采用动态规划的方法来分析这个题，
如果正确直接即可。经过数学证明可以知道，越早发射代价最高的火箭，代价越小。

证明 ：
设 Wi, Wj 为第i个火箭和第j个火箭延误一小时的代价，Ti，Tj，为原定发射时间，K1，K2，为火箭延误后的发射时间，我们这里不妨设K1 < K2，且可以知道的是，调换这两个火箭的发射顺序，对其他火箭并没有任何影响。

我们用 P1 表示先发射火箭i再发射火箭j造成的损失，P2表示先发射火箭j再发射火箭i造成的损失

P1 = (K1 - Ti) * Wi + (K2 - Tj) * Wj

P2 = (K1 - Tj) * Wj + (K2 - Ti) * Wi

要是 P1 < P2 我们有 (K1 - Ti) * Wi + (K2 - Tj) * Wj < (K1 - Tj) * Wj + (K2 - Ti) * Wi

化简可以得到 K1(Wi - Wj) < K2(Wi - Wj)

已知的是K1 < K2， 那么我只需要Wi > Wj 即可满足越早发射损失越小的目标。这与原定发射时间没有任何关系，我们只需要把火箭每小时损失费用从大到小排列一遍，再用简单的贪心即可计算出来。

接下来就是n < k 的情况了：

我们可以发现，如果我们还是按照上一种情况来计算这种情况的话，我们无法确定发射时间比原定时间晚。这怎么办呢？只能动态计算所有火箭的损失。边读取边计算即可保证火箭延误后的发射时间比原定时间晚。维护一个优先队列，在第k+i时刻，如果此时在原定计划有未发射的火箭，那么先读入他，入队，去掉队首元素，更新答案，知道第k+n时刻，所有火箭发射完毕。

实现的过程中需要对结构题进行排序，在结构体中重载运算符号即可~
