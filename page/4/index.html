C5-小面包

Posted on 2019-11-24 | In 第五次上机题解 |

题面

又要发小面包了。这次我们有许多3* 6的小面包和6* 6的方糕，以及一个6* N的长方形盒子，强迫症的某楠一定要把它们整齐的装到盒子里，并且要尽量装满。请问有多少总不同装法？

输入

对于每组数据，输出一行，为最终计算对1000007取模得到的结果。

输出

输出一行，能处理的垃圾最大重量，保留2位小数，单位为吨

输入样例

输出样例

Hint

输入为3时，只能放入一块小面包。

输入为6时，有三种情况：

（1）竖着放两块小面包

（2）横着放两块小面包

（3）放一块方糕

分析题目

这个题其实挺难想的，我想错了好久。

可以看到的是想要递推得到 f[i] 的答案，无非是从 f[i-1] 和 f[i-2] 递推而来，但是结尾不同可能会造成不同的答案。

比如结尾是 一个3*6的小面包的时候，想要递推得到下一个答案，可以将上一个小面包倒置过来横着放，又是一种不同的答案。

那么我们可以用f1[i] 表示 最后一个是竖着放置的小面包的方法数目，f2[i]表示不是竖着放置的小面包的数目。那么有递推式：

f1[i] = f1[i-1] + f2[i-1]

f2[i] = f1[i-2] + f2[i-2] + f1[i-1]

将初始化命名为
f1[1] = 1, f1[2] = 1;f2[1] = 0, f2[2] = 2;即可得到正确答案，想一想为什么…

讲上式简化可以得到f[i] = f[i-1] + 2 * f[i-2] 的递推式。

C5-图3

Posted on 2019-11-23 | In 第五次上机题解 |

题面

一个无向图，N个点编号1~N。M条边，每条边有一个权值c。

问对于每条边，最少删除多少条边后，可以使得存在一个最小生成树包含这条边。

输入

第一行两个正整数N、M。(2≤N,M≤100)

接下来M行，每行三个整数abc，表示ab之间存在一条权值为c的边。（1≤a,b≤N，1≤c≤500）

输出

输出一行M个数，数之间用空格隔开

输入样例

输出样例

分析题目

对于每一条边而言，只保留小于它的权值的边，讲流量设置为1，跑一边最大流即可。

原理可以根据最小生成树的Kruskal算法，即从小到大枚举边的权值，如果两个端点i,j在一个联通块里面就pass，否则将两个联通块相连。

建图的时候只保留比这一条边的边权小的边，两个端点为汇点和源点，跑一遍最大流即可。

其实意思就是，你想要最小生成树含有这个边，肯定要删除大于等于零个比这个边的边权小的边，这个时候其实比这个边小的边权都已经是等价的了。这时候将这些边建图，求他们的最小割，最小割是什么意思呢，就是最大流。为什么要求最小割呢，原因是，在建树的过程当中，如果在遍历到这条边的时候，两个端点i，j，已经在一个联通块里面，可以预见的是最小生成树不会包含这条办。那么如何让最小生成树含有这条边呢，自然是将拆开这两个点，将所有权值小于这条边的权值的边能够连通的通路全部删除，即实现一遍最大流即可。当然删除的时候是有选择的删除一些边，最小的删除的边数就是最大流。

比如考虑一个这样的深度神经网络： s 3 4 t

将与s相连的3条边删掉即可实现在运行Kruskal算法的时候不会将 s t相连。
