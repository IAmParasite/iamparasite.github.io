C4-非负权单源最短路

Posted on 2019-11-09 | In 第四次上机题解 , 模板 | | 阅读数

题面

给一个n(1<=n<=2500)个点m(1<=m<=6200)条边的无向图，求s到t的最短路。

输入

第一行四个由空格隔开的整数n,m,s,t 之后的m行，每行三个正整数a、b、c，表示一条从a到b长度为c的边。

输出

一个整数表示从s到t的最短路长度。数据保证至少存在一条道路。

输出样例

分析题目

先附上大佬来链接：https://www.cnblogs.com/bcoier/p/10363284.html

然后分析一下这个题目：

最短路的算法主要有3个：floyd(O(n^3))、Dijkstra(O(nlogn))、SPFA(O(KE))（k是进队列次数）

既然这个题这么水，俺的题解可不能这么就水过去了（哗哗哗）

Floyd算法

我们首先理解（背）一下Floyd算法的板子,Floyd的时间复杂度是O(n^3)，可以计算任意两个点之间的最短路径，所以你要求单源最短路径，并且只有两个点之间的最短距离，这个解法是肯定肯定不能过的：

试着将邻接矩阵修改成为邻接表…然而失败了，Floyd 的思想就是运用的动态规划的思想。

Dj算法算是比较容易理解得算法之一了,仅仅使用于权值为正的情况：

邻接表法:

值得注意的是，贪心的思路是到源点的最小值，而不是上一条边的最小值。

我们可以注意到的是，我们每搜索一个点，都要遍历一遍最小的边，那么我可以用数据结构来优化这个结构，我们可以想到的是用优先队列来使最小的元素在队头即可。时间优化为O(nlogn)

代码略丑，可以看大佬博客：https://blog.csdn.net/AAMahone/article/details/90145830

SPFA算法

考场毒瘤SPFA算法：时间复杂度为O(km)(k为入队次数)
SPFA算法可以用于负权值的图，最坏情况下退化到O(nm)

彩蛋

SPFA是写最短路径而不用堆优化的唯一的人。他身材很高大；青白脸色，皱纹间时常夹些伤痕；
一部乱蓬蓬的花白的胡子。穿的虽然是女装，可是又脏又破，似乎十多年没有补，也没有洗。
他对人说话，总是满口O(kE)，叫人半懂不懂的。因为他姓S，别人便从描红纸上的“Shortest Path Faster Algorithm”这半懂不懂的话里，替他取下一个绰号，叫作SPFA。

SPFA一到机房，所有写代码的人便都看着他笑，有的叫道，“SPFA，你又TLE了！”他不回答，对我说，“打1e5个结点，要2e5条边。”便排出一条队列。
他们又故意的高声嚷道，“你一定又被出题人卡了！”SPFA睁大眼睛说，“你怎么这样凭空污人清白……”“什么清白？我前天亲眼见你被出题人卡到O(nm)，吊着打。”
SPFA便涨红了脸，额上的青筋条条绽出，争辩道，“TLE不能算O(nm)……O(nm)！卡常数的事，能算O(nm)么？”接连便是难懂的话，什么“SPFA的复杂度是O(kE)”，什么“可以证明k一般小于等于2”之类。
引得众人都哄笑起来；机房内外充满了快活的空气。

有些最短路模板不是使用邻接矩阵和邻接表实现的，而是用前向星来实现~
