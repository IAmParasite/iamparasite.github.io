C3-排座位

Posted on 2019-11-01 | In 第三次上机题解 | | 阅读数

题面

将n个女生和m个男生排成一列，要求是连续的女生不能超过x，连续的男生不能超过y
请问：一共有多少种排座位的方式（结果对1000007取模）。

注：所有的女生看作相同，所有男生看做相同。

注意内存限制。

输入

多组数据输入，数据组数小于20

每组数据包括四个正整数n、m、x、y（1≤n,m≤2000,1≤x,y≤2000），含义见描述

输出

对于每组数据，输出一行，为排座位的方法数（结果对1000007取模）

输出样例

Hint

1表示女生，0表示男生

第一组数据，方法只有一种，即：010

第二组数据，方法有五种，即：01011，01101，10101,10110，11010

分析题目

用dp[i][j][0]和dp[i][j][1]分别表示排列i个女生j个男生时末尾为男生的情况。

多维动归显然不行，我们注意到，连续不能超过x个女生的情况，只需要在男生后面依次“填充”min(i,x)个女生即可，根本不需要数组记录末尾连续的人数，同理男生也是一样，在女生后面“填充”min(j,y)个男生即可。

值得注意的是，dp[0][0][1] = dp[0][0][0] = 1，这个初始化好像有悖逻辑，其实不然，我们同样需要更新的是i/j中带一个0情况，为防止错误，我们对带0的情况，也进行遍历，既然规则都是一样的，我们便可以一起遍历。

想想怎么样可以再优化一下呢？我们不妨用笔计算一下：

假设min(i,x) = 3， min(j,y) = 4

计算i = 5, j = 6 情况的时候你可以注意到
dp[5][6][0] = dp[4][6][1] + dp[3][6][1] + dp[2][6][1]

dp[5][6][1] = dp[5][5][0] + dp[5][4][0] + dp[5][3][0] + dp[5][2][0]

同样的进行下一次遍历(j++)的时候，计算i = 5, i = 7 的时候

dp[5][7][0] = dp[4][7][1] + dp[3][7][1] + dp[2][7][1]

dp[5][7][1] = dp[5][6][0] + dp[5][5][0] + dp[5][4][0] + dp[5][3][0]

我们可以发现规律：当i 相同的时候，以男生结尾的情况有重叠的部分，且相加的个数好像是确定的，个数就是min(j,y)个。同样当j相同的时候，以女生结尾的部分也是重复的，只要维护两个数组sum_1[i]，sum_2[j]来表示需要相加的“前缀和”，得出答案再更新，就可以再O(n^2)完成。TLE的点肯定是，x，y比较大的时候，这时的算法效率为O(n^3)，很容易被卡超时。
